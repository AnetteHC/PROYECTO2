Funciones de un Servidor Web


La interacción entre un servidor y sus usuarios se fundamenta en el uso de HTTP, el protocolo de transferencia de hipertexto o su versión cifrada, HTTPS. Para entender su operación, es vital comprender que el servidor web permanece en constante espera de solicitudes de información.


Es importante tener en mente que cada dispositivo, ya sea una computadora, teléfono inteligente o tableta, posea una dirección IP única e invariable que lo diferencia de otros en la red. De este modo, el servidor web puede enviar la información necesaria que el usuario está esperando.


En este contexto, para que el servidor web cumpla con su rol, requiere recibir una petición desde un navegador, es decir, una solicitud transmitida desde una dirección IP hacia la dirección IP del servidor que almacena los archivos del sitio en cuestión.


A continuación, el servidor web busca en sus archivos la información requerida, interpreta las líneas de código correspondientes y envía el resultado al navegador cuya dirección IP originó la solicitud. Este resultado se muestra a los usuarios, siendo un proceso constante durante la navegación en sitios web. Una vez finalizado este procedimiento, podemos afirmar que el servidor web ha cumplido su cometido.
